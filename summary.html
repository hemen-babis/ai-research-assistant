Summary

- The project report for CS 314 in Fall 2024 should include two main components: a project retrospective and presentation slides.
- The project retrospective should cover the entire term, discussing both the successes and failures of the project.
- For issues that did not go well, the retrospective should provide 1 or 2 suggestions for improvement.
- It should also include lessons learned and reflections on what would be done differently in future projects.
- The length of the retrospective should be between 2 and 3 pages.
- The presentation slides, intended for stakeholders, should number between 3 and 4.
- These slides should detail the project's completed functionality, testing methods, and any known defects or issues.
